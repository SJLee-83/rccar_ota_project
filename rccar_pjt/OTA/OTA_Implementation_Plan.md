# Dual Bank OTA 업데이트 구현 계획 (OTA 폴더 내 독립 구현)

사용자님의 정확한 의도에 맞추어, 외부 폴더(`OTA_RA6E1`, `Infotainment_GUI` 등)의 기존 코드를 일절 참고하거나 가져오지 않고, **오직 `OTA` 폴더 내에 있는 파일들만 수정**하여 5단계 OTA 시나리오를 완벽하게 구현하는 계획입니다.

## 📌 목표 (User Requirements)
1. **Dual Bank 메모리 사용**: 뱅크 0과 뱅크 1을 번갈아가며 스왑(Swap) 업데이트
2. **초기 상태**: 뱅크 0에서 `version 0.0` 실행
3. **다운로드 중지**: OTA 다운로드 진행 중에는 RC카 모터 동작(차량 조종) 불가 처리
4. **펌웨어 교체**: 다운로드가 완벽하게 끝나면 뱅크 1을 부팅 영역으로 변경하여 `version 0.1` 실행

---

## 🚀 구체적 실행 계획 (Only in `OTA/` directory)

### STEP 1. Python GUI (송신부)
- **대상 파일**: `OTA/mainwindow.py`
- **구현 내용**:
  - 외부 코드를 복사하지 않고, `btn_select_bin`, `btn_start_ota` 등 기존에 만들어진 UI 버튼의 이벤트 핸들러를 새롭게 작성합니다.
  - 선택한 `.bin` 파일을 읽어들이고, 256바이트 단위로 자릅니다(Chunk).
  - Paho-MQTT를 이용해 `OTA/Data` 토픽으로 조각들을 연속 전송하고, GUI의 프로그레스 바(Progress Bar)를 실시간으로 업데이트합니다.

### STEP 2. ESP32 (중계부)
- **대상 파일**: `OTA/esp32.ino`
- **구현 내용**:
  - 기존 폴링 통신 구조를 유지하되, `OTA/Command`와 `OTA/Data` 토픽을 새롭게 구독합니다.
  - OTA 명령이 들어오면 일반 RC 조종 명령 처리를 잠시 멈추고, 수신된 펌웨어 조각(Chunk)들을 SPI를 통해 RA6E1로 그대로 넘겨주는(Bypass) 로직을 추가합니다.

### STEP 3. 르네사스 RA6E1 (수신 및 Dual Bank 제어부)
- **대상 파일**: `OTA/hal_entry.c`
- **구현 내용**:
  - `hal_entry.c` 파일 하나 안에 **Flash HP 드라이버 제어 로직**을 직접 구현합니다.
  - **다운로드 중지**: SPI로 `OTA_START` 명령을 받으면 모터 속도를 0으로 멈추고, 플래시 메모리(비활성 뱅크)를 Erase(지우기)합니다. 이 기간 동안 RC 조종 명령은 무시됩니다.
  - **펌웨어 라이트**: 이어지는 펌웨어 조각들을 비활성 뱅크 주소에 `R_FLASH_HP_Write`를 사용해 기록합니다.
  - **펌웨어 교체**: `OTA_END` 명령을 받으면, `R_FLASH_HP_StartUpAreaSelect`를 호출하여 뱅크 0과 1을 스왑한 뒤 `NVIC_SystemReset()`으로 재부팅하여 새로운 버전을 실행합니다.
